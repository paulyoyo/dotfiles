# dotfiles

Personal macOS dotfiles — zsh + starship + kitty + Terminal.app + zellij, all themed with **Salvaje**, a warm dark palette built from five anchor colors.

Managed as a **bare git repository** in `$HOME/.dotfiles` with the working tree pointed at `$HOME`. No symlinks, no `stow`, no framework — just git.

## What's in here

```
.zshrc                                main zsh config — starship + plugins + aliases
.zprofile                             login shell env — brew shellenv + Python PATH
.zshenv                               cargo env
.secrets.example                      template for ~/.secrets (API keys)
.gitignore                            bare-repo allow-list
README.md                             this file

.config/kitty/kitty.conf              kitty — minimal, includes Salvaje theme
.config/kitty/Salvaje.conf            kitty Salvaje color theme

.config/starship.toml                 starship prompt (24-bit RGB — for kitty)
.config/starship-terminal.toml        starship prompt (16-color ANSI — for Terminal.app)

.config/terminal-profiles/
  Salvaje.terminal                    macOS Terminal.app profile (import manually)

.config/zellij/config.kdl             zellij main config (theme = salvaje)
.config/zellij/themes/salvaje.kdl     zellij Salvaje theme
.config/zellij/layouts/default.kdl    default layout with zellaude tab bar
```

**Not tracked** (machine-local):
- `~/.secrets` — API keys, tokens, anything sensitive
- `~/.config/zellij/plugins/*.wasm` — binary plugins (self-update from upstream)
- `~/.config/zellij/plugins/zellaude-hook.sh` — auto-generated by the plugin
- `~/.config/zellij/plugins/zellaude.json` — per-machine plugin state

## Prerequisites

### macOS packages (Homebrew)

```bash
# Install Homebrew first if needed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Core
brew install git zsh starship

# CLI tools referenced by .zshrc
brew install fzf eza bat fd direnv rbenv jq

# Optional (zellaude clickable notifications)
brew install terminal-notifier

# Zellij + kitty
brew install zellij
brew install --cask kitty

# Nerd Font used by every terminal
brew install --cask font-meslo-lg-nerd-font
```

### Other

- **NVM** for Node: <https://github.com/nvm-sh/nvm>
- **rbenv** Ruby versions — installed at `/opt/rbenv` (adjust `RBENV_ROOT` in `.zshrc` if different)
- **Rust** (only if building zellaude from source): <https://rustup.rs>

### Fonts

Two font families are used:

1. **MesloLGS Nerd Font Mono** — fallback for icon glyphs (starship OS icons, LazyVim file-tree icons, zellaude activity symbols, braille art). Install via Homebrew cask (already in the list above).

2. **GT Pressura LCGV Mono** — primary code/terminal font. **Commercial font from [Grilli Type](https://www.grillitype.com/typeface/gt-pressura). NOT tracked in this public repo due to licensing.** On a new machine you must install it yourself:
   ```bash
   # Copy your licensed .ttf files into ~/Library/Fonts/
   cp /path/to/licensed/GT-Pressura-LCGV-Mono-*.ttf ~/Library/Fonts/
   ```
   Required files at minimum:
   - `GT-Pressura-LCGV-Mono-Regular.ttf` (PostScript: `GTPressuraLCGVMono-Regular`)
   - `GT-Pressura-LCGV-Mono-Bold.ttf`
   - `GT-Pressura-LCGV-Mono-Regular-Italic.ttf`
   - `GT-Pressura-LCGV-Mono-Bold-Italic.ttf`

   Without these files, kitty and Terminal.app will fall back to whatever macOS substitutes. Functional but not matching the intended look.

## Install on a new machine

### 1. Clone the bare repo

```bash
git clone --bare https://github.com/paulyoyo/dotfiles.git $HOME/.dotfiles
```

### 2. Define the `config` alias

```bash
alias config='/opt/homebrew/bin/git --git-dir=$HOME/.dotfiles --work-tree=$HOME'
```

### 3. Hide untracked files

```bash
config config --local status.showUntrackedFiles no
```

### 4. Checkout

```bash
config checkout
```

If this fails because an existing file would be overwritten (e.g. the default `~/.zshrc` from a fresh macOS install), back them up:

```bash
mkdir -p ~/.config-backup
config checkout 2>&1 | grep -E "^\s+\." | awk '{print $1}' | \
  xargs -I{} mv {} ~/.config-backup/

config checkout
```

### 5. Create `~/.secrets` from the template

```bash
cp ~/.secrets.example ~/.secrets
chmod 600 ~/.secrets
$EDITOR ~/.secrets   # fill in ANTHROPIC_API_KEY and any others
```

### 6. Reload zsh

```bash
exec zsh
```

At this point kitty + starship + zsh are fully configured. A few extras still need machine-specific setup below.

## Post-install

### Import Terminal.app profile

```bash
open ~/.config/terminal-profiles/Salvaje.terminal
```

This opens a new Terminal.app window with the Salvaje profile imported. Then set it as default:

- Terminal.app → **Settings** → **Profiles** → select **Salvaje** → click **Default**

Or via command line:

```bash
defaults write com.apple.Terminal "Default Window Settings" -string "Salvaje"
defaults write com.apple.Terminal "Startup Window Settings" -string "Salvaje"
killall cfprefsd
```

Then **Cmd+Q** and relaunch Terminal.app.

### Download the zellaude plugin

```bash
mkdir -p ~/.config/zellij/plugins
curl -sL --fail -o ~/.config/zellij/plugins/zellaude.wasm \
  https://github.com/ishefi/zellaude/releases/latest/download/zellaude.wasm
```

The plugin self-installs its Claude Code hook on first load — no further action needed. Start a new zellij session (`zellij` in a fresh kitty window) and the zellaude bar appears at the top.

### Symlink kitty CLI (optional)

```bash
ln -sf ~/Applications/kitty.app/Contents/MacOS/kitty ~/.local/bin/kitty
ln -sf ~/Applications/kitty.app/Contents/MacOS/kitten ~/.local/bin/kitten
```

(Adjust `~/Applications` to `/Applications` if you installed kitty there.)

## Day-to-day workflow

The `config` alias is the only thing you use. It behaves exactly like `git`, operating only on tracked dotfiles.

```bash
# See what changed
config status

# Stage a change
config add .zshrc

# Commit
config commit -m "zshrc: add alias"

# Push
config push

# Pull updates on another machine
config pull
```

### Tracking a new file

Every file in `$HOME` is ignored by default. To track a new one, add it to `.gitignore` as an **un-ignore** rule, then `config add -f`:

```bash
# Example: track ~/.config/nvim/init.lua
echo '!.config/nvim/' >> ~/.gitignore
echo '!.config/nvim/init.lua' >> ~/.gitignore
config add -f .gitignore .config/nvim/init.lua
config commit -m "track nvim init.lua"
```

### Never accidentally commit secrets

The `.secrets` file is listed under `# Never commit secrets` in `.gitignore`. Keep all API keys, tokens, and credentials there — the dotfiles repo will never see them. When onboarding a new machine, only `.secrets.example` is in the repo; you recreate `.secrets` manually.

## Terminal matrix

| Terminal | Font | Theme source | Starship config | Notes |
|---|---|---|---|---|
| **kitty** | GT Pressura LCGV Mono 18pt + MesloLGS NF fallback via `symbol_map` | `kitty/Salvaje.conf` | `starship.toml` (24-bit RGB) | Primary terminal. Nerd Font glyph ranges routed to MesloLGS NF automatically. |
| **Terminal.app** | GTPressuraLCGVMono-Regular 18pt + native macOS font substitution | `Salvaje.terminal` profile | `starship-terminal.toml` (16-color ANSI) | Falls back to named ANSI for colors. Terminal.app has no symbol_map so icon glyphs rely on macOS native font fallback — install MesloLGS NF too so macOS picks it as fallback. |
| **zellij** | (inherits from host terminal) | `zellij/themes/salvaje.kdl` | — | Layout includes zellaude activity bar |

`.zshrc` automatically switches `$STARSHIP_CONFIG` based on `$TERM_PROGRAM`:

```zsh
if [[ "$TERM_PROGRAM" == "Apple_Terminal" ]]; then
  export STARSHIP_CONFIG="$HOME/.config/starship-terminal.toml"
else
  export COLORTERM=truecolor
fi
```

## Starship

The prompt is a 1:1 port of the upstream [Gruvbox Rainbow preset](https://starship.rs/presets/gruvbox-rainbow) with only the 10-color palette swapped for Salvaje equivalents. The format string, module configs, OS symbols, language modules, and separator glyphs are all byte-identical to upstream.

Palette mapping (for easy diffing against upstream):

| Gruvbox variable | Gruvbox hex | Salvaje hex | Salvaje name |
|---|---|---|---|
| `color_fg0` | `#fbf1c7` | `#F7F7F5` | Bright Snow |
| `color_bg1` | `#3c3836` | `#120E15` | Near-black |
| `color_bg3` | `#665c54` | `#2A1A38` | Lilac-grey |
| `color_blue` | `#458588` | `#8956BA` | Deep Lilac |
| `color_aqua` | `#689d6a` | `#6AAFB8` | Teal |
| `color_green` | `#98971a` | `#8FBF6C` | Sage |
| `color_orange` | `#d65d0e` | `#C599E2` | Wisteria |
| `color_purple` | `#b16286` | `#A474D4` | Bright Lilac |
| `color_red` | `#cc241d` | `#E0637A` | Rose |
| `color_yellow` | `#d79921` | `#EAD82F` | Golden Glow |

Segment order (left → right): OS icon + user → directory → git branch/status → language runtimes (c, cpp, rust, go, node, php, java, kotlin, haskell, python) → docker/conda/pixi → command duration + time → prompt character on the next line.

## zellij + zellaude

Zellij runs with the Salvaje theme and the [zellaude](https://github.com/ishefi/zellaude) plugin replacing the default tab bar. zellaude shows live Claude Code activity per session:

| Symbol | Meaning |
|---|---|
| ◆ | Session starting |
| ● | Thinking |
| ⚡ | Running bash |
| ◉ | Reading / searching files |
| ✎ | Editing / writing files |
| ⊜ | Spawning subagent |
| ◈ | Web search / fetch |
| ⚙ | Other tool |
| ▶ | Waiting for prompt |
| ⚠ | Waiting for permission (flashes + desktop notification) |
| ✓ | Done |
| ○ | Idle |

Click any tab to switch; click a `⚠` tab to jump straight to the pane asking for permission.

## Uninstall

Remove the bare repo and the config alias. Tracked files stay in `$HOME`; delete them manually if you want a clean slate.

```bash
rm -rf ~/.dotfiles
unalias config
```

To uninstall zellaude:

```bash
rm -f ~/.config/zellij/plugins/zellaude.wasm
rm -f ~/.config/zellij/plugins/zellaude-hook.sh
# Then manually remove the zellaude hook entry from ~/.claude/settings.json
```
